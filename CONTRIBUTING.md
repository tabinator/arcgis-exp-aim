# Contributing

## Branches

- `main` is the stable branch.
- Use feature branches named like `feat/aim-package-form`, `fix/map-selection`, or `chore/ci-baseline`.

## Commits

Use concise conventional-style commit messages:

```text
feat: add AiM package creation form
fix: handle missing map widget
chore: initialize repo workflow
```

## Pull Requests

Every PR should include:

- What changed
- How it was tested
- Risks or follow-up work
- ArcGIS Experience Builder deployment notes, when relevant

## Local Checks

Run the checks that exist for the current repo state. As widget source is added, CI should evolve to include TypeScript, linting, and package/build verification.
